MIKROTIK ROUTEROS

Mengonfigurasi mode wireless AP dan Station
Konfigurasi jaringan nirkabel untuk akses point dan client station

Topologi jaringan di bawah akan digunakan sebagai dasar ilustrasi untuk konfigurasi mode wireless AP dan Station di MikroTik. Dalam topologi ini, satu MikroTik Router bertindak sebagai Access Point (AP) dan satu atau lebih MikroTik Router lainnya bertindak sebagai Station yang terhubung ke AP tersebut.

Router MikroTik pertama akan berfungsi sebagai AP yang memancarkan sinyal WiFi, sementara router-router MikroTik lainnya akan berfungsi sebagai Station yang menerima dan terhubung ke sinyal WiFi dari AP. Ini adalah konfigurasi umum untuk komunikasi nirkabel antar-router dan penyediaan akses nirkabel untuk perangkat client.

Berikut adalah komponen-komponen utama dalam topologi ini:

1. MikroTik Router (AP)
# Router MikroTik yang dikonfigurasi sebagai Access Point (mode ap-bridge)
2. MikroTik Router (Station)
# Router MikroTik yang dikonfigurasi sebagai Station/Client (mode station)
3. SSID dan Security Profile
# SSID dan konfigurasi keamanan untuk koneksi wireless

Pastikan menggunakan enkripsi yang kuat (WPA2 atau WPA3) untuk melindungi jaringan wireless Anda dari akses tidak sah. Hindari penggunaan WEP atau WPA1 yang sudah diketahui memiliki kelemahan keamanan.

1. MODE ACCESS POINT (AP)

Pengertian Mode Access Point

Dari sumber MikroTik Documentation (2024), mode Access Point (AP) pada MikroTik adalah mode operasi wireless dimana router berfungsi sebagai titik akses yang memancarkan sinyal WiFi yang dapat diakses oleh perangkat client. Mode ini memungkinkan MikroTik menjadi pusat koneksi wireless untuk berbagai perangkat seperti laptop, smartphone, atau router lain yang dikonfigurasi sebagai station.

Dari sumber MikroTik Wiki (2023), RouterOS mendukung dua mode AP utama: mode ap-bridge yang memungkinkan hingga 2007 klien terhubung secara bersamaan, dan mode bridge yang berfungsi sebagai bridge layer 2 sederhana. Untuk sebagian besar kasus penggunaan umum, mode ap-bridge adalah pilihan yang direkomendasikan.

Konfigurasi Mode AP menggunakan CLI

Dari sumber NetworkLessons (2024), konfigurasi mode AP pada MikroTik memerlukan beberapa langkah utama, yaitu mengatur interface wireless ke mode ap-bridge, menentukan SSID, band, frekuensi, dan security profile.

Dari sumber MikroTik Wiki (2023), langkah-langkah konfigurasi dasar mode AP meliputi pengaturan mode, SSID, frequency, band, dan security profile. Semua parameter ini dapat dikonfigurasi melalui CLI MikroTik dengan perintah-perintah yang tepat.

Berikut adalah langkah-langkah konfigurasi mode AP melalui CLI:

# 1. Melihat interface wireless yang tersedia
/interface wireless print
# Output contoh:
Flags: X - disabled, R - running
0 X name="wlan1" mtu=1500 mac-address=48:A9:8A:XX:XX:XX arp=enabled
# 2. Mengatur mode AP, SSID, dan band
/interface wireless set wlan1 mode=ap-bridge ssid="MikroTik-AP" band=2ghz-b/g/n
# 3. Mengatur frequency (channel)
/interface wireless set wlan1 frequency=2437
# 4. Membuat security profile
/interface wireless security-profiles
add name="profile1" mode=dynamic-keys authentication-types=wpa2-psk wpa2-pre-shared-key="password123"
# 5. Menerapkan security profile ke interface wireless
/interface wireless set wlan1 security-profile="profile1"
# 6. Mengaktifkan interface wireless
/interface wireless enable wlan1

Verifikasi Konfigurasi AP

Setelah mengkonfigurasi mode AP, penting untuk memverifikasi bahwa konfigurasi tersebut berjalan dengan baik. Berikut adalah beberapa perintah untuk memeriksa status AP:

# Memeriksa status interface wireless
/interface wireless print
# Memeriksa client yang terhubung ke AP
/interface wireless registration-table print
# Output contoh jika ada client terhubung:
# MAC-ADDRESS INTERFACE SIGNAL... UPTIME
0 48:8F:5A:XX:XX:XX wlan1 -65dBm... 3h32m10s
# Memeriksa status access list
/interface wireless access-list print

Konfigurasi Tambahan untuk AP

Dari sumber MikroTik Documentation (2024), ada beberapa parameter tambahan yang dapat dikonfigurasi untuk mengoptimalkan kinerja AP:

# Membatasi jumlah klien yang dapat terhubung
/interface wireless set wlan1 max-station-count=30
# Mengatur lebar channel (untuk throughput lebih tinggi)
/interface wireless set wlan1 channel-width=20/40mhz-Ce
# Mengatur country untuk mengikuti regulasi frekuensi lokal
/interface wireless set wlan1 country=indonesia
# Mengatur distance untuk optimasi performa
/interface wireless set wlan1 distance=indoors

Pengaturan distance mempengaruhi algoritma ACK timing. Untuk area indoor, gunakan nilai "indoors". Untuk koneksi jarak jauh, gunakan nilai numerik dalam kilometer (misalnya "10" untuk 10km) atau "dynamic" agar MikroTik secara otomatis menyesuaikan timing berdasarkan client terjauh.

Kesimpulan Mode Access Point: Dari 2 pendapat tersebut dapat disimpulkan bahwa mode Access Point (AP) pada MikroTik adalah konfigurasi yang memungkinkan router memancarkan sinyal WiFi yang dapat diakses oleh perangkat lain. Konfigurasi dasar mode AP melibatkan pengaturan mode ap-bridge, SSID, frequency, band, dan security profile, yang semuanya dapat dikonfigurasi dengan mudah melalui CLI. Dengan konfigurasi yang tepat, AP MikroTik dapat melayani banyak klien secara bersamaan dan menyediakan konektivitas wireless yang andal.

2. MODE STATION

Tampilan topologi di gns3

Pengertian Mode Station

Dari sumber MikroTik Documentation (2024), mode Station pada MikroTik adalah mode operasi wireless dimana router berperan sebagai client yang terhubung ke access point lain. Dalam mode ini, MikroTik akan mencari dan terhubung ke SSID yang dikonfigurasi, berfungsi sebagai "client wireless" atau "station" yang menerima sinyal dari access point.

Dari sumber System Zone (2021), MikroTik dalam mode Station dapat digunakan sebagai "WiFi Client" untuk menghubungkan jaringan lokal ke jaringan lain melalui koneksi wireless. Mode ini sering digunakan dalam skenario WISP (Wireless Internet Service Provider) di mana router MikroTik terhubung ke hotspot atau AP dari ISP sebagai client.

Tipe-tipe Mode Station

Dari sumber MikroTik Wiki (2023), RouterOS mendukung beberapa tipe mode station dengan karakteristik berbeda:

station # Mode station standar, tidak mendukung bridging L2
station-wds # Mode untuk koneksi point-to-point, bekerja hanya dengan AP MikroTik
station-pseudobridge # Mendukung bridging L2 terbatas dengan translasi MAC address
station-bridge # Mendukung transparent bridging, hanya berfungsi dengan AP MikroTik

Dari sumber Wireless Station Modes Documentation (2024), pemilihan tipe mode station yang tepat tergantung pada kebutuhan bridging L2. Jika bridging L2 tidak diperlukan (seperti pada jaringan yang di-routing), mode station standar adalah pilihan terbaik karena paling efisien.

Konfigurasi Mode Station menggunakan CLI

Berikut adalah langkah-langkah konfigurasi mode Station melalui CLI:

# 1. Melihat interface wireless yang tersedia
/interface wireless print
# 2. Mengatur mode Station, SSID, dan band
/interface wireless set wlan1 mode=station ssid="MikroTik-AP" band=2ghz-b/g/n
# 3. Membuat security profile (jika belum ada)
/interface wireless security-profiles
add name="station_profile" mode=dynamic-keys authentication-types=wpa2-psk wpa2-pre-shared-key="password123"
# 4. Menerapkan security profile ke interface wireless
/interface wireless set wlan1 security-profile="station_profile"
# 5. Mengaktifkan interface wireless
/interface wireless enable wlan1

Untuk mode station dengan bridging (misalnya station-bridge atau station-pseudobridge):

# Menggunakan mode station-pseudobridge untuk bridging terbatas
/interface wireless set wlan1 mode=station-pseudobridge ssid="MikroTik-AP"
# Atau menggunakan mode station-bridge untuk transparent bridging (hanya dengan AP MikroTik)
/interface wireless set wlan1 mode=station-bridge ssid="MikroTik-AP"
# Jika ingin menambahkan interface wireless ke bridge
/interface bridge add name=bridge1
/interface bridge port add bridge=bridge1 interface=wlan1
/interface bridge port add bridge=bridge1 interface=ether1

Verifikasi Konfigurasi Station

Berikut adalah beberapa perintah untuk memeriksa status koneksi Station:

# Memeriksa status interface wireless
/interface wireless print
# Memeriksa status koneksi wireless
/interface wireless monitor wlan1
# Output contoh jika terkoneksi:
status: connected-to-ess
band: 2ghz-b/g/n
frequency: 2437MHz
ssid: MikroTik-AP
signal-strength: -67dBm
# Melakukan scan untuk melihat AP yang tersedia
/interface wireless scan wlan1

Konfigurasi IP pada Station

Setelah koneksi wireless berhasil, perlu dilakukan konfigurasi IP agar perangkat dapat berkomunikasi di jaringan:

# Mengatur IP address statis
/ip address add address=192.168.0.2/24 interface=wlan1
# Atau menggunakan DHCP client untuk mendapatkan IP otomatis
/ip dhcp-client add interface=wlan1 disabled=no
# Mengatur default route (jika diperlukan)
/ip route add gateway=192.168.0.1
# Memeriksa konektivitas dengan ping
/ping 192.168.0.1

Ketika menggunakan mode station, pastikan SSID dan password yang dikonfigurasi sudah benar. Jika menggunakan enkripsi WPA2/WPA3, pastikan parameter security-profile sudah sesuai dengan konfigurasi AP yang dituju.

Kesimpulan Mode Station: Dari 2 pendapat tersebut dapat disimpulkan bahwa mode Station pada MikroTik berfungsi sebagai client wireless yang mencari dan terhubung ke access point lain. MikroTik menyediakan beberapa tipe mode station dengan karakteristik berbeda yang dapat dipilih sesuai kebutuhan, terutama terkait kemampuan bridging L2. Konfigurasi mode station memerlukan pengaturan mode, SSID, band, dan security profile yang sesuai dengan AP tujuan, serta konfigurasi IP yang tepat untuk komunikasi jaringan.

3. KONFIGURASI LANJUTAN DAN TROUBLESHOOTING

Menggunakan Mode Kombinasi AP dan Station

Dari sumber RouterOS Documentation (2024), MikroTik memungkinkan penggunaan satu perangkat untuk beroperasi sebagai AP dan Station secara bersamaan dengan membuat interface virtual. Fitur ini sangat berguna untuk mengimplementasikan wireless repeater atau memperluas jangkauan jaringan wireless.

Dari sumber MikroTik Wiki (2023), konfigurasi wireless repeater dapat dilakukan dengan mengatur interface utama sebagai station dan membuat interface virtual sebagai AP. Kedua interface tersebut kemudian dihubungkan melalui bridge untuk meneruskan traffic.

# 1. Mengkonfigurasi interface utama sebagai station
/interface wireless set wlan1 mode=station-pseudobridge ssid="Uplink-AP" security-profile="station_profile"
# 2. Membuat interface virtual AP
/interface wireless add master-interface=wlan1 name=virtual-AP ssid="Extended-Network" mode=ap-bridge security-profile="profile1"
# 3. Membuat bridge
/interface bridge add name=bridge1
# 4. Menambahkan interface ke bridge
/interface bridge port add bridge=bridge1 interface=wlan1
/interface bridge port add bridge=bridge1 interface=virtual-AP
# 5. Mengaktifkan interface
/interface wireless enable wlan1
/interface wireless enable virtual-AP

Troubleshooting Koneksi Wireless

Dari sumber MikroTik Troubleshooting Guide (2024), masalah koneksi wireless sering disebabkan oleh ketidakcocokan parameter, interferensi, atau masalah konfigurasi security profile. Berikut adalah beberapa perintah untuk diagnosa masalah wireless:

# Melihat log wireless
/log print where topics=wireless
# Melakukan scan frekuensi untuk melihat channel yang sibuk
/interface wireless frequency-scan wlan1 duration=10s
# Melihat status dan signal strength dari koneksi
/interface wireless monitor wlan1 once
# Memeriksa parameter pada kedua perangkat (AP dan station)
/interface wireless print detail
# Melihat klien yang terdaftar pada AP
/interface wireless registration-table print

Dari sumber Network Troubleshooting Best Practices (2023), beberapa langkah pemecahan masalah yang efektif meliputi:

# Reset konfigurasi wireless jika diperlukan
/interface wireless reset-configuration wlan1
# Mengubah channel untuk menghindari interferensi
/interface wireless set wlan1 frequency=auto
# Memeriksa power output
/interface wireless set wlan1 tx-power=20
# Mematikan fitur advanced yang mungkin menyebabkan masalah
/interface wireless set wlan1 wmm-support=disabled country=no_country_set

Tips Optimasi Performa

Beberapa tips untuk mengoptimalkan performa koneksi wireless:

# Mengatur antenna-gain jika menggunakan antena eksternal
/interface wireless set wlan1 antenna-gain=12
# Mengoptimalkan penggunaan chain
/interface wireless set wlan1 chains=0,1
# Mengatur DFS untuk menghindari interferensi radar (pada band 5GHz)
/interface wireless set wlan1 dfs-mode=no-radar-detect
# Mengatur country sesuai regulasi lokal
/interface wireless set wlan1 country=indonesia

Penggunaan parameter tx-power yang terlalu tinggi tidak selalu menghasilkan koneksi yang lebih baik. Hal ini dapat menyebabkan interferensi dan penurunan throughput. Sesuaikan parameter ini berdasarkan jarak dan kondisi lingkungan.

Kesimpulan Konfigurasi Lanjutan dan Troubleshooting: Dari 2 pendapat tersebut dapat disimpulkan bahwa MikroTik menyediakan kemampuan untuk mengkonfigurasi perangkat sebagai AP dan Station secara bersamaan melalui interface virtual, yang sangat berguna untuk implementasi wireless repeater. Untuk troubleshooting, RouterOS menyediakan berbagai alat diagnosa seperti frequency-scan, monitor, dan log yang dapat membantu mengidentifikasi dan menyelesaikan masalah koneksi wireless. Optimasi performa dapat dilakukan dengan penyesuaian parameter seperti channel, tx-power, antenna-gain, dan pengaturan lainnya sesuai dengan kondisi lingkungan.
